# Security Policy

## Supported Versions

Snablr is currently maintained on the latest `main` branch and current tagged releases.

Security fixes are prioritized for:

- the latest release
- the current development branch

Older releases may not receive backported fixes.

## Reporting A Vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

Instead:

1. open a private security advisory in GitHub, if enabled for the repository
2. or contact the maintainer through a private channel listed on the project profile

When reporting, include:

- affected version or commit
- impact summary
- reproduction steps
- any relevant logs or screenshots with secrets redacted

## Scope

This policy applies to:

- the Snablr CLI
- bundled rule packs
- report generation
- release artifacts published from this repository

## Safe Disclosure Expectations

Please avoid:

- public proof-of-concept release before a fix is available
- including real credentials, customer data, or private infrastructure details in reports
- submitting unsafe or non-synthetic fixtures to the repository

## Operational Safety

Snablr is intended for authorized defensive security work only. Use it only against systems and data you own or are explicitly permitted to assess.
